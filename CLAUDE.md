## ­¤ōÉ Architecture and Structure

The codebase follows a clear separation of concerns between mathematical computation and IO handling:

1.  **Algorithm Core (`PerlinNoise`):**
    *   Located in `PerlinNoise/src/`.
    *   Contains the static method `GenerateNoise(int Width, int Height)`, which computes raw noise data (as a flat array of floats) based on input dimensions. This is the mathematical engine of the project.

2.  **IO and Serialization Layer (`Image`):**
    *   Located in `PerlinNoise/src/`.
    *   Handles image file management, abstracting away format details. The implementation focuses heavily on saving to **PPM** format (`SavePPMImage`).
    *   It consumes the raw float buffer generated by `PerlinNoise` via `CopyBuffer()`.

3.  **Orchestration (Entry Point):**
    *   The main application logic resides in `src/main.cpp`. This file demonstrates the complete pipeline: initialize dimensions $\rightarrow$ call `PerlinNoise::GenerateNoise()` $\rightarrow$ pass result to `Image` object $\rightarrow$ save image via `testImage.SaveImage()`.
